# 모듈 3 - 랩 1 - 연습 1 - 클라우드용 Microsoft Defender 활성화

## 랩 시나리오

여러분은 클라우드용 Microsoft Defender로 클라우드 워크로드 보호를 구현하고 있는 회사에서 근무하고 있는 보안 운영 분석가입니다.  이 랩에서는 클라우드용 Microsoft Defender를 활성화합니다.


### 작업 1: Azure Portal에 액세스하여 구독 설정

이 연습에서는 이 랩과 이후 랩을 완료하는 데 필요한 Azure 구독을 설정합니다.

1. WIN1 가상 머신에 Admin으로 로그인합니다. 암호로는 **Pa55w.rd**를 사용합니다.  

2. Microsoft Edge 브라우저를 엽니다. 이미 열려 있다면 새 탭을 엽니다.

3. Edge 브라우저에서 Azure Portal(https://portal.azure.com) 로 이동합니다.

4. 랩 호스팅 공급자가 제공한 관리자 사용자 이름용 테넌트 전자 메일 계정을 복사하여 **로그인** 대화 상자에 붙여 넣은 후 **다음**을 선택합니다.

5. 랩 호스팅 공급자가 제공한 관리자의 테넌트 암호를 복사하여 **암호 입력** 대화 상자에 붙여 넣은 후 **로그인**을 선택합니다.

6. Azure Portal의 검색 창에 *구독*을 입력하고 **구독**을 선택합니다. 

7. *"Azure Pass - Sponsorship"* 구독이 표시되면(또는 선택한 언어로 된 해당 이름) 작업 2로 진행합니다. 그렇지 않은 경우 강사에게 테넌트 관리자 사용자 자격 증명을 사용하여 Azure 구독을 만드는 방법을 물어보세요. **참고:** 구독 만들기 프로세스는 10분 정도 걸릴 수 있습니다. 

>**중요:** 이러한 랩은 강의 중에 USD $10 미만의 Azure 서비스를 사용하도록 설계되었습니다.


### 작업 2: Log Analytics 작업 영역 만들기

이 작업에서는 클라우드용 Microsoft Defender와 함께 사용할 Log Analytics 작업 영역을 만듭니다.

1. Azure Portal의 검색 창에 *Log Analytics*를 입력하고 **Log Analytics 작업 영역**을 선택합니다.

2. 명령 모음에서 **+만들기**를 선택합니다.

3. 리소스 그룹의 **새로 만들기**를 선택합니다.

4. *RG-Defender*를 입력하고 **확인**을 선택합니다.

5. 이름에 대해서는 *uniquenameDefender*와 같은 고유 이름을 입력합니다.

6. **검토 + 만들기**를 선택합니다.

7. 작업 영역 유효성 검사에 통과하면 **만들기**를 선택합니다. 새 작업 영역이 프로비전될 때까지 기다립니다. 몇 분 정도 걸릴 수 있습니다.


### 작업 3: 클라우드용 Microsoft Defender 활성화

이 작업에서는 클라우드용 Microsoft Defender를 활성화하고 구성합니다.

1. Azure Portal의 검색 창에 *Defender*를 입력하고 **클라우드용 Microsoft Defender**를 선택합니다.

2. **시작** 페이지에서 **업그레이드** 섹션으로 이동하여 구독이 선택되어 있는지 확인하고 페이지 아래쪽에서 **업그레이드** 단추를 선택합니다. *평가판 시작됨* 알림이 나타날 때까지 기다립니다. **힌트:** 상단 표시줄에서 종 모양 단추를 클릭하여 Azure Portal 알림을 검토합니다.

3. 구독에 이미 포함되어 있는 가상 머신에 에이전트를 설치하는 옵션이 다음 페이지에 표시됩니다. **여기에서는 아무것도 하지 않습니다**.

4. 포털 메뉴의 관리 영역에서 **환경 설정**을 선택합니다.

5. **"Azure Pass - Sponsorship"** 구독(또는 선택한 언어로 이에 해당하는 이름)을 선택합니다. 

6. *모든 클라우드용 Microsoft Defender 요금제 활성화* 아래에서 활성화된 다양한 기능을 그리고 *다음을 위한 Microsoft Defender* 열 아래에서 보호되는 Azure 리소스를 검토합니다.

7. 설정 영역에서 **자동 프로비전**을 선택합니다.

8. 자동 프로비전 - 확장을 검토합니다. **Azure VM용 Log Analytics 에이전트**가 **꺼짐**으로 설정되어 있는지 확인합니다.

9. 페이지 오른쪽 위에서 'x'를 선택하여 설정 페이지를 닫고 **환경 설정**으로 다시 돌아서 구독 왼쪽의 '>'을 선택합니다.

10. 앞에서 만든 Log Analytics 작업 영역 *uniquenameDefender*를 선택하고 사용 가능한 옵션과 가격을 검토합니다. **모든 클라우드용 Microsoft Defender 계획 사용**을 선택하고 **저장**을 선택합니다.


### 작업 4: 온-프레미스 서버에 Azure Arc 설치

온-프레미스 서버에 Azure Arc를 설치하면 온-프레미스 서버를 더 쉽게 온보딩할 수 있습니다.  Azure Arc를 설치하면 Azure에서 온-프레미스 서버를 관리할 수 있도록 설정됩니다.

이 작업에서는 온-프레미스 서버에 Azure Arc를 설치합니다.

1. WINServer 가상 머신에 Administrator로 로그인합니다. 암호로는 **Passw0rd!** 를 사용합니다.  

2. Microsoft Edge 브라우저를 열고 Azure Portal(https://portal.azure.com)로 이동합니다.

3. 랩 호스팅 공급자가 제공한 **테넌트 전자 메일** 계정을 복사하여 **로그인** 대화 상자에 붙여 넣은 후 **다음**을 선택합니다.

4. 랩 호스팅 공급자가 제공한 **테넌트 암호**를 복사하여 **암호 입력** 대화 상자에 붙여넣은 후 **로그인**을 선택합니다.

5. Azure Portal의 검색 창에 *Arc*를 입력하고 **Azure Arc**를 선택합니다.

6. 탐색 창의 **인프라** 아래에서 **서버**를 선택합니다.

7. **+ 추가**를 선택합니다.

8. "단일 서버 추가" 섹션에서 **스크립트 생성**을 선택합니다.

9. **다음**을 선택하여 리소스 세부 정보 탭으로 이동합니다.

10. 앞에서 만든 리소스 그룹을 선택합니다. 힌트: *RG-Defender*

    >**참고:** 아직 리소스 그룹을 만들지 않았으면 다른 탭을 열고 리소스 그룹을 만든 후 시작하세요.

11. *서버 세부 정보* 및 *네트워크 연결성* 옵션을 검토합니다. **다음**을 선택하여 태그 탭으로 이동합니다.

12. **다음**을 선택하여 스크립트 다운로드 및 실행 탭으로 이동합니다.

13. **등록**을 선택합니다.

    >**참고:** 처리가 완료되도록 3분 이상 기다리세요.

14. 아래로 스크롤하고 **다운로드** 단추를 선택합니다. 힌트: 브라우저에서 다운로드를 차단하는 경우 브라우저에서 다운로드를 허용하도록 설정합니다. Edge 브라우저에서 점 3개 "..."를 선택하고 **유지**를 선택합니다. 

15. Windows 시작 단추를 마우스 오른쪽 단추로 클릭하고 **Windows PowerShell (Admin)** 을 선택합니다.

16. 사용자 이름을 입력하라는 메시지가 표시되면 Administrator를 입력합니다.

17. 암호를 입력하라는 메시지가 표시되면 Passw0rd!를 입력합니다.

18. cd C:\Users\Administrator\Downloads를 입력합니다.

19. *Set-ExecutionPolicy -ExecutionPolicy Unrestricted*를 입력하고 Enter 키를 누릅니다.

20. 모두 예에 해당하는 **A** 키를 누른 다음 Enter 키를 누릅니다.

21. *[.\OnboardingScript.ps1]* 을 입력하고 Enter 키를 누릅니다.

22. **R** 키를 눌러 한 번 실행하고 Enter 키를 누릅니다(몇 분 정도 걸릴 수 있음).

23. PowerShell 출력 마지막 줄에 있는 지침을 따라 디바이스 등록을 완료합니다.  여기에는 브라우저를 통한 디바이스 인증이 포함됩니다.  url(https://microsoft.com/devicelogin) 을 복사하고 새 Edge 브라우저 탭에 입력합니다. Windows PowerShell 창으로 돌아가고, 인증할 코드를 복사하여 앞서 열어 놓은 탭에 붙여넣고, **다음**을 선택합니다. 테넌트 관리자 계정을 선택하고 *Azure Connected Machine Agent에 로그인하려고 하십니까?* 창에서 **계속**을 선택합니다. 

24. *"리소스를 Azure에 온보딩함"* 메시지가 Windows PowerShell 창에 나타나면 스크립트를 다운로드했던 Azure Portal 페이지로 이동하여 **닫기**를 선택합니다. **Azure Arc로 서버 추가**를 닫아서 Azure Arc **서버** 페이지로 돌아갑니다.

25. WINServer 서버 이름이 표시될 때까지 **새로 고침**을 선택합니다.

    >**참고:** 몇 분 정도 걸릴 수 있습니다.


### 작업 5: 온-프레미스 서버 보호

이 작업에서는 Windows Server에 필요한 에이전트를 수동으로 설치합니다.

1. **클라우드용 Microsoft Defender**로 이동하여 **시작** 페이지를 선택합니다.

2. **시작** 탭을 선택합니다.

3. 아래로 스크롤하고 *Azure가 아닌 서버 추가* 섹션 아래에서 **구성**을 선택합니다.

4. 앞에서 만든 작업 영역 옆의 **업그레이드**를 선택합니다.  시간이 몇 분 걸릴 수도 있습니다. *"작업 영역에 대한 Defender 요금제가 저장되었습니다"* 알림이 표시될 때까지 기다립니다.

5. 앞에서 만든 작업 영역 옆의 **+ 서버 추가**를 선택합니다.

6. **Windows 에이전트 다운로드(64비트)** 를 선택합니다.

7. 다운로드한 파일을 실행합니다.

8. **에이전트 설정 옵션** 마법사 페이지가 나타날 때까지 **다음**을 선택하고, **에이전트를 Azure Log Analytics(OMS)에 연결**을 선택한 후에 **다음**을 선택합니다.

9. **작업 영역 ID** 및 **기본 키**를 Azure Portal에서 복사하여 마법사 페이지의 해당 필드에 붙여넣고 **다음**을 선택합니다.

10. 설치를 계속 진행합니다. 완료되면 **마침**을 선택합니다.

11. "클라우드용 Microsoft Defender" 포털로 이동하고 **재고**를 선택합니다.

12. 5분 이내에 WINServer가 나타납니다. 표시하기 위해 **새로 고침**을 선택해야 할 수도 있습니다.

13. 다음 랩으로 이동하고 나중에 돌아와서 **클라우드용 Microsoft Defender**의 **재고** 섹션을 검토할 수 있습니다.
